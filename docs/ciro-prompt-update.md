# CIRO UI 텍스트 및 프롬프트 (2024)

## 1. 헬스체크 UI 텍스트

### 메인 타이틀
```
오늘의 건강 상태
```

### 설명 텍스트
```
오늘 나의 기분 또는 팀의 건강 상태를 1~10점으로 체크해보세요.
매일 기록하면 나와 팀의 변화를 한눈에 볼 수 있어요.
```

### 슬라이더 레이블
```
현재 점수: {healthScore}점
```

### 점수 가이드
- **1~3점**: 😞 컨디션이 좋지 않아요
- **4~6점**: 😐 평범한 하루예요
- **7~9점**: 😊 좋은 상태예요
- **10점**: 🎉 최고예요!

### 저장 완료 메시지 (콘솔)
```
Health check saved successfully
```

### 에러 메시지 (콘솔)
```
Failed to save health check: {error}
```

## 4. 챗봇 시스템 메시지

### 초기 인사 (템플릿별)

#### STAR 템플릿
```
안녕하세요! STAR 방식으로 경험을 정리해보겠습니다.
Situation(상황), Task(과제), Action(행동), Result(결과) 순서로 질문드릴게요.
```

#### 5F 템플릿
```
안녕하세요! 5F(사실-느낌-발견-향후-피드백) 방식으로 회고를 진행합니다.
편안하게 답변해주세요.
```

#### KPT 템플릿
```
안녕하세요! KPT(Keep-Problem-Try) 회고를 시작합니다.
잘했던 점, 문제점, 시도할 점을 함께 정리해봐요.
```

#### 4L 템플릿
```
안녕하세요! 4L(Liked-Learned-Lacked-Longed for) 회고입니다.
좋았던 것, 배운 것, 부족했던 것, 바라는 것을 나눠볼게요.
```

#### PMI 템플릿
```
안녕하세요! PMI(Plus-Minus-Interesting) 방식으로 진행합니다.
긍정적인 면, 부정적인 면, 흥미로운 점을 찾아봐요.
```

## 9. 설문 질문 텍스트

### 질문 1: 회고 목적
```
이번 회고의 주요 목적은 무엇인가요?
```

선택지:
- 📊 성과 분석 및 평가
- 🎓 학습과 성장
- 🔧 문제 해결 및 개선
- 💡 아이디어 발굴

---

### 질문 2: 프로젝트 규모
```
회고할 경험의 규모는 어느 정도인가요?
```

선택지:
- 👤 개인 프로젝트
- 👥 소규모 팀 (2-5명)
- 🏢 중규모 팀 (6-15명)
- 🌐 대규모 조직 (16명 이상)

---

### 질문 3: 소요 시간
```
회고에 할애할 수 있는 시간은?
```

선택지:
- ⚡ 10분 이내 (빠른 회고)
- ⏱️ 20-30분 (적당한 회고)
- 📅 30분 이상 (깊이 있는 회고)

---

### 질문 4: 선호 방식
```
어떤 방식의 회고를 선호하시나요?
```

선택지:
- 📋 구조화된 틀 (체계적)
- 💭 자유로운 형식 (유연함)
- 🎯 핵심 집중형 (간결함)
